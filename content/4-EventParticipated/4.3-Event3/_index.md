# Harvest Report: "First Cloud Journey Community Day (06/06/2026)"

### Event Information
*   **Event Name:** First Cloud Journey (FCJ) Community Day
*   **Event Date:** June 6, 2026
*   **Location:** 36th Floor, Bitexco Financial Tower, No. 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City
*   **Organizers:** Team AWS G3 in collaboration with the First Cloud Journey Community

### Event Objectives
*   **Program Goal:** To share practical knowledge on career roadmaps to become Cloud, Security, and DevOps Engineers; update participants on new technologies like Machine Learning (ML), Docker, and AI (GraphRAG).
*   **Value for Attendees:** Foundations to advanced concepts about cloud infrastructure, gaining real-world experience from speakers currently working at large enterprises.


### Event Highlights
*   **Network Security with ML:** Introduced a Machine Learning-based Network Intrusion Detection System (NIDS) integrated with AWS WAF to overcome traditional ruleset limitations. The solution uses the CSE-CIC-IDS2018 dataset for real-world model training.
*   **Containerization:** Introduction to Docker, comparing VMs and Containers, highlighting Docker's lightweight nature and its *"Build once, run anywhere"* capability.
*   **Career Roadmap:** Shared a personal journey from IT Helpdesk to Senior Sysadmin, emphasizing that Linux and Networking are essential foundations.
*   **Multiplayer in the Cloud:** Used AWS WebSockets, Lambda, and DynamoDB to connect Godot game clients for real-time data communication.
*   **Artificial Intelligence (AI):** Built a GraphRAG application using Amazon Bedrock and Amazon Neptune to optimize LLM information retrieval through graph-based relationships.
*   **Soft Skills:** *"The Art of Effective Teamwork"* with 4 golden rules and tools like Trello and Slack.

### Key Learnings
*   **Mindset & Methodology:** Cloud is not just a technology but a new way of thinking (Cloud Mindset) featuring automation, elastic scaling, and pay-as-you-go pricing models.
*   **Technical Knowledge:** Methods to handle imbalanced data in ML, basic Docker CLI commands, and real-time WebSockets architecture on AWS.
*   **Best Practices:**
    *   Never test on production environments to maintain system availability and business credibility.
    *   Automate repetitive tasks and maintain thorough documentation.
    *   Data quality is vital for ML model performance.

### Application to My Work
*   **Current Project:** Utilize Docker to standardize development and deployment environments (CI/CD). Deploy AWS WAF to protect web applications against vulnerabilities like SQL Injection or XSS.
*   **Technologies to Explore:** Adopt Amazon Bedrock for GenAI features or AWS GameLift for managing dedicated game servers.
*   **Workflow Improvement:** Build a real-time monitoring dashboard to enhance threat visibility.

### Event Experience
*   **Practice & Demos:** Attended live demos showing the NIDS system, Docker CLI operations, and real-time Godot game client synchronization via AWS.
*   **Networking:** Connected with speakers via LinkedIn QR codes to exchange professional guidance.
*   **Most Memorable Aspect:** The powerful synergy between AWS cloud services and modern technologies like ML/AI to solve practical security and operational challenges.

### Key Takeaways
*   **Most Critical Knowledge:** Signature-based security is insufficient against zero-day exploits; ML-driven security is necessary.
*   **Real-world Experience:** Starting point does not matter; consistency and hands-on practice on real projects matter more than just collecting certifications.
*   **Future Development Plan:** Focus deeply on 1-2 core skills before expanding, building a personal portfolio with practical projects.
