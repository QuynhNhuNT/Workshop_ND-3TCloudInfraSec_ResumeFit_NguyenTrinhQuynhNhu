# Harvest Report: "FCAJ Community Day (23/05/2026)"

### Event Information
*   **Event Name:** FCAJ Community Day
*   **Event Date:** Weekend morning (May 23, 2026)
*   **Location:** 36th Floor, Bitexco Financial Tower, No. 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City (In-Person Event)
*   **Organizers:** AWS Study Group in collaboration with the FCAJ Community

### Event Objectives
*   **Program Goal:** Not just to listen to seminars, but to serve as a hub to connect, network, and inspire one another (with the chance of meeting potential partners or like-minded peers).
*   **Key Messages:** Centered around topics spanning AI, Cloud, and real-world stories drawn from the experiences of industry experts. Discussions included Prompt Optimization, AWS technologies (like CloudFront), the Hackathon journey, and the mindset for building business-grade AI architecture (Enterprise grade).
*   **Value for Attendees:** Help attendees understand job market dynamics under the impact of AI, how to build core foundations to prevent obsolescence, and equip them with problem-solving mindsets for real-world enterprise challenges.

### Event Highlights
*   **Problem Overview:** AI is reducing the cost of software development, which will cause a massive surge in software demand. This will create many new jobs in operations (Platform engineering) and debugging (fixing AI-generated code). Another issue is that generating code using AI without proper context produces bloated, low-quality outputs (LLM hallucinations).
*   **Solutions Introduced:** 
    *   Set clear system prompts and context (Context/System Prompt) including: Goal, Persona, and Format instead of asking vague questions.
    *   Use a Multi-Agent System (orchestration of multiple agents) to solve complex tasks (e.g., credit analysis for Startups) rather than stuffing everything into a single AI model.
*   **Technologies, Services, & Tools:**
    *   **AWS:** CloudFront (Flat-rate pricing, content compression, DDoS protection), AWS Bedrock, EC2, Lambda, VPC.
    *   **AI/LLM:** Amazon Agent platform, Obsidian (for building a Second Brain), MCP (Model Context Protocol).
    *   **Other:** Terraform (Infrastructure as Code - IaC).
*   **Demos & Case Studies:** 
    *   *UTM Morpho Project:* Demoed a hackathon tool that uses 3 AI agents to generate HTML/UI from images, allowing users to drag and drop or edit layouts directly on the interface.
    *   *LLM Parameters Demo:* Ran comparisons with Temperature = 0 on AWS Bedrock vs. Local to show that cloud-based AI outputs still display variance due to inference optimization mechanisms.
*   **Key Takeaways:** In enterprise environments, security compliance and risk assessment (Guardrails, Security, Audit) are critical—far more important than simply connecting numerous AI tools.

### Key Learnings
*   **Mindset & Methodology:** LLM is a "Probabilistic engine" (probability-based model), so do not assume it will always return the exact same output even when setting Temperature = 0. Do not let AI go on complete auto-pilot; always keep humans in the loop (Human in the loop).
*   **Technical Knowledge:** Understand how CloudFront optimizes costs via flat-rate pricing to prevent "billing shock" caused by high traffic spikes or DDoS attacks.
*   **Best Practices:** Avoid the "Internet puller" syndrome (blindly copying libraries or code snippets from the internet without deep comprehension of the context).
*   **Real-world Insights:** In a Hackathon, having too many ideas (over-generation) wastes time. Limit the scope and focus on solving one core problem (such as enhancing the editing experience) to complete the prototype within the 36-hour limit.

### Application to My Work
*   **Current Project Applicability:** Design AI systems based on Multi-agent principles: assign specific tasks to specialized agents (e.g., Financial Analysis Agent, Market Research Agent). Implement mTLS and VPC Origin to secure resources fully.
*   **Technologies to Explore:** Use Terraform to deploy infrastructure as code (IaC) for easy version management. Test edge protection features of AWS CloudFront (geo-blocking, automatic bot blocking).
*   **Workflow Improvement Ideas:** Instead of downloading data manually, integrate AI Agents with tools (such as using MCP to connect Excel) to automate dashboard creation or meeting summaries.

### Event Experience
*   **Speaker Insights:** Realized that in professional environments, employers do not just ask for demos but probe deeply into the practicality of the "Product" and its business use cases.
*   **Hands-on Experience:** Witnessed a live demo of a product built in 36 hours and saw a visual illustration comparing LLM behavior locally vs. on the cloud.
*   **Networking:** Encouraged self-confidence and public speaking, highlighting the fierce competition in the job market.
*   **Most Memorable Aspect:** The strong warning about directly pasting ChatGPT-generated code into enterprise systems, which can degrade coding standards and introduce severe security holes.

### Key Takeaways
*   **Most Important Concept:** Core foundations (Foundation), university degrees, and basic software engineering skills are still mandatory. AI is a bonus, not a replacement for core software engineering principles (such as secure password encoding/decoding).
*   **Real-world Experience:** To convince executives to approve budgets for AI projects, you must clearly demonstrate ROI and concrete financial savings—*"Numbers speak louder than words"*.
*   **Future Development Plan:** Dive deeper into AI Mindset and AI Adoption. Keep in mind the system design rule: It must not only run, but *"run securely, run reliably, and serve the user"*. Read official documentation carefully when configuring AI parameters instead of relying on online chatter.
